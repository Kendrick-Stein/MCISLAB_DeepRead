## 论文点评

### SpatialEvo
- **Title**: SpatialEvo: Self-Evolving Spatial Intelligence via Deterministic Geometric Environments
- **Institutes**: Zhejiang University, StepFun
- **Source**: [link](https://arxiv.org/abs/2604.14144) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-SpatialEvo]]
- **核心**: 提出用 Deterministic Geometric Environment (DGE) 替代 model consensus 做 self-evolving 训练。核心 insight 是 3D 空间推理的答案可以从点云和 camera pose 中确定性计算，不需要模型投票来构造伪标签。单模型分饰 questioner 和 solver 两角，在 DGE 的"零噪声"奖励信号下通过 GRPO 做 online RL，配合 task-adaptive scheduler 自动涌现 curriculum。9 个 benchmark 上 3B 和 7B 均达最高平均分。
- **锐评**: 
  - **核心 insight 干净且正确**——3D 几何的确定性属性确实是 self-evolving 的天然优势。w/o Physical Grounding 的 ablation 最直接有力：VSI-Bench 从 46.1 暴跌到 18.8，这个 27+ 的差距一锤定音地证明了 DGE 的价值。
  - **"零噪声"是自欺欺人**：DGE 的第一阶段 entity parsing 靠 GPT-OSS-120B 从自由文本中抽取结构化实体。LLM 会 hallucinate、会抽错——这叫把噪声从 model voting 转移到了 LLM parsing pipeline。
  - **适用边界极窄**：只能在 ScanNet 类室内静态场景跑，要求"high-quality indoor point cloud reconstructions + calibrated camera poses"。室外场景？动态场景？直接不可用。
- **Rating**: `3` 🔥 ，核心 insight 正确、ablation 有力，但适用场景极窄

### RAGEN-2
- **Title**: RAGEN-2: Reasoning Collapse in Agentic RL
- **Institutes**: Northwestern University, Stanford University, Microsoft, UIUC, Imperial College London, Oxford University, University of Washington
- **Source**: [link](https://arxiv.org/abs/2604.06268) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-RAGEN2]]
- **核心**: 多轮 Agent RL 训练中存在 "template collapse" 现象——模型输出的 reasoning chain 在单个 input 内部看起来很多样（高 entropy），但跨 input 之间其实是同一个模板，对 input 不敏感。作者把 reasoning quality 拆成两个维度：within-input diversity（条件熵）和 cross-input distinguishability（互信息），提出 MI proxy 做在线诊断。进一步用 SNR 机制解释：reward variance 低的时候 task gradient 弱，input-agnostic 的 regularization 项占据主导。提出 SNR-Aware Filtering——每轮用 reward variance 做代理筛选高 signal 的 prompt 做 update。
- **锐评**: 
  - **发现本身有价值**——template collapse 确实是个真实存在的 failure mode，entropy 监控的盲区点得很准，MI 分解也是教科书级的 correct。
  - **但方法端偏弱**：(1) MI proxy 本质就是 in-batch cross-scoring，信息论包装了一下但实现极其朴素；(2) SNR-Aware Filtering 说白了就是 "挑 reward variance 高的 batch 训练"，一个 heuristic filter 被写成了一整个 section 的 formal proof；(3) 缺少具体数字——从摘要到 introduction 看不到任何一个具体的 performance improvement 百分比。
- **Rating**: `2` 👀 ，发现 template collapse 和 MI 诊断框架有参考价值，但解决方案只是一个 reward variance filter

### Claw-Eval
- **Title**: Claw-Eval: Toward Trustworthy Evaluation of Autonomous Agents
- **Institutes**: PKU, HKU 等
- **Source**: [link](https://arxiv.org/abs/2604.06132) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-ClawEval]]
- **核心**: 指出当前 agent benchmark 三大缺陷（只判结果不看过程、安全评估不足、模态覆盖窄），提出 Claw-Eval：300 个任务覆盖 9 类，每个 action 通过 execution trace + audit log + environment snapshot 三通道记录，实现 trajectory-aware grading。评分协议包含 Completion / Safety / Robustness 三维度。14 个前沿模型测试发现：黑盒评估漏掉 **44% 安全违规**和 13% 鲁棒性失败。
- **锐评**: 
  - **44% safety violation miss rate** 这个数字确实扎眼，是全文最有说服力的 claim。
  - **本质是 benchmark engineering**——trajectory-aware grading、多试验取统计、多维度评分，每个 component 单独看都不是新 idea，价值在于把它们组合起来并提供一个统一框架。
  - **定位问题**："toward trustworthy" 这个 framing 有点 overclaim——你只是做了更细致的评分，离 "trustworthy" 差了 formal verification、out-of-distribution generalization 好几层。
- **Rating**: `2` 👀 ，数据点有价值（44% miss rate 值得记住），本质是 benchmark engineering

### HybridMemory WM
- **Title**: Out of Sight but Not Out of Mind: Hybrid Memory for Dynamic Video World Models
- **Institutes**: HUST & Kuaishou Technology
- **Source**: [link](https://arxiv.org/abs/2603.25716) 📰 HF Daily **📒 论文笔记**: [[Papers/2603-HybridMemory]]
- **核心**: 现有 video world model 的 memory 机制把世界当静态画布，动态主体出画再入画时往往冻结、扭曲或消失。提出 Hybrid Memory 范式，要求模型同时记住静态背景 + 追踪动态主体的隐含轨迹。构建 HM-World 数据集（UE5 渲染的 59K 视频），提出 HyDRA 方法：用 3D 卷积把 memory latent 压缩成 token，再用 spatiotemporal affinity 做 Top-K 检索替代标准 self-attention。
- **锐评**: 
  - **问题提得不错**：现有 memory 确实忽略动态主体的独立运动逻辑，exit-entry 场景是合理的压力测试。
  - **但整篇论文是在 UE5 沙盒里自导自演**：数据集是自己渲染的、方法是只在自己数据上训的、评估也是在自己数据上评的。这跟真实世界的复杂度差了至少两个数量级。
  - **Baseline 太弱**：只比了 Baseline（Wan2.1 + camera encoder）、DFoT、Context-as-Memory，全都 retrain 在 HM-World 上。
  - **增益数字经不起细看**：Subject Consistency 0.903 → 0.926、Background Consistency 0.925 → 0.932，全是小数点后两位的边际提升。
- **Rating**: `2` 👀 ，问题意识到位、方法设计合理，但实验闭环太封闭

### MultiWorld
- **Title**: MultiWorld: Scalable Multi-Agent Multi-View Video World Models
- **Institutes**: The University of Hong Kong, Sreal AI
- **Source**: [link](https://arxiv.org/abs/2604.18564) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-MultiWorld]]
- **核心**: 把 Wan2.2-5B 改造成 multi-agent multi-view 视频 world model。MACM 用 RoPE（base frequency 从 10000 降到 20）给每个 agent action 加身份编码；GSE 用冻结的 VGGT 提取 3D-aware 全局状态保证多视角一致。在 ItTakesTwo 多人游戏和 RoboFactory 多机械臂场景上验证。
- **锐评**: 
  - **工程拼接，不算 breakthrough**——本质是 Wan2.2 + frozen VGGT 的缝合，架构上没什么理论创新。
  - **游戏场景数字好看，机器人场景尴尬**——Concat-View 在 Action (92.0 vs 88.7) 和 RPE (0.82 vs 1.52) 上双杀 MultiWorld，论文只用一句"仅训练 2 view 不可直接比"糊弄过去。
  - **Scalability 全靠嘴**——标题大写 SCALABLE，但实验只做了 2-agent（游戏）和最多 4-robot，真正的 scalability 验证在哪？
  - **缺少下游验证**——world model 最大的价值是当 simulator 用，但这篇只报了 FVD/PSNR 这些 generation metrics，没有下游 task 结果。
- **Rating**: `2` 👀 ，问题定义对了，但 robot 场景被 baseline 双杀不解释

### HiVLA
- **Title**: HiVLA: A Visual-Grounded-Centric Hierarchical Embodied Manipulation System
- **Institutes**: HKU, Shanghai AI Lab, SJTU, CUHK
- **Source**: [link](https://arxiv.org/abs/2604.14125) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-HiVLA]]
- **核心**: 提出 hierarchical VLA 框架，用 Qwen3-VL 8B 做高层 planner（task decomposition + visual grounding 输出 bbox），用 DiT action expert 做低层控制，通过 cascaded cross-attention 桥接。在 RoboTwin 上达到 83.3% 总平均成功率，比 H-RDT 高 12.7%。
- **锐评**: 
  - **亮点**：Tab.2 的 bbox noise robustness 实验——100% bbox noise 下仍有 57% 成功率，证明 policy 不会盲目跟随 planner 的错误输出。
  - **硬伤一**：Real-world 实验只有 demo 视频，零量化数据。
  - **硬伤二**：Baseline 选择有严重 cherry-picking 嫌疑——DexGraspVLA、HiRobot 全被以各种理由排除。
  - **硬伤三**：Marginal contribution 没有 claim 说的那么大——Ours w/o Skill（只有 global + local）也有 78.7%，真正的 grounding 机制价值被"总提升"数字放大了。
- **Rating**: `2` 👀 ，工程上扎实的 hierarchical VLA 实现，但 baseline 选择性缺席、real-world 零量化

### Agent-World
- **Title**: Agent-World: Scaling Real-World Environment Synthesis for Evolving General Agent Intelligence
- **Institutes**: Renmin University of China, ByteDance Seed
- **Source**: [link](https://arxiv.org/abs/2604.18292) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-AgentWorld]]
- **核心**: 从 MCP servers、工具文档、工业 PRD 采集数千主题，用 deep-research agent 自主挖掘数据库和工具接口，构建 1,978 个环境 / 19,822 个工具。通过工具图游走 + programmatic 双轨合成可验证任务，再用 GRPO 多环境 RL 训练。在 23 个 benchmark 上超过 EnvScaler、AWM 等 environment scaling 基线。
- **锐评**: 
  - **规模确实大**——1,978 个环境 + 19,822 个工具在 environment synthesis 赛道里算头部的。环境数从 10 到 2000 的 scaling 曲线清晰，+20.1pt 的平均增益有说服力。
  - **"Real-World"水分很大**——所有环境本质都是 JSON/CSV 文件的本地读写，没有 live API、没有网络请求。
  - **MCP-Mark 绝对分数暴露问题**——8B 模型只有 8.9%，14B 也只有 13.3%。号称 outperform，但连 20% 都没到。
  - **Self-evolving 的"self"要打引号**——diagnosis agent 用的是 GPT-OSS-120B，这本质上是 distillation，不是 self-evolution。
- **Rating**: `2` 👀 ，工程量大、scaling 曲线扎实，但 MCP-Mark 绝对分数低得离谱

### Agent Trajectory Retrieval
- **Title**: Learning to Retrieve from Agent Trajectories
- **Institutes**: Renmin University of China, Chinese Academy of Sciences
- **Source**: [link](https://arxiv.org/abs/2604.04949) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-AgentTrajectory]]
- **核心**: 提出 agent 搜索行为与人类不同，应从 agent trajectory 中提取监督信号训练 retriever。方法 LRAT：browse 行为当 positive，unbrowsed 当 negative，reasoning length 当 relevance intensity 权重。在 6 个 agent + 2 个 retriever 上验证，success rate 提升 5%~38%。
- **锐评**: 
  - **方向是对的**——"agent 取代人类成为 retriever 的消费者"这个 observation 有价值。
  - **但硬伤不少**：(1) 全部 trajectory 来自单一 agent Tongyi-DeepResearch-30B，泛化性完全未知；(2) E5-Large 上只有 +5%~17% 的提升，对强 retriever 增益明显缩水；(3) Wiki-25-Dump 11.2M chunks 离真实 web search 差了三个数量级。
- **Rating**: `2` 👀 ，方向值得跟进，但实验设计和 claim 之间的 gap 还需要更多证据

### Video-MME-v2
- **Title**: Video-MME-v2: Towards the Next Stage in Benchmarks for Comprehensive Video Understanding
- **Institutes**: CASIA, Tencent, Nanyang Technological University 等
- **Source**: [link](https://arxiv.org/abs/2604.05015) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-VideoMMEv2]]
- **核心**: 现有视频 benchmark 饱和，搞了个新 benchmark：三层渐进能力层级（信息聚合 → 时序建模 → 复杂推理）+ group-based 非线性评分（$(N/4)^2$ 二次惩罚 + first-error truncation），800 视频 3200 题。Gemini-3-Pro 49.4 被人类 90.7 碾压，发现 thinking mode 在纯视觉场景反而退化。
- **锐评**: 
  - **Group-based 非线性评分**这个设计确实比逐题 accuracy 更有信息量，能暴露模型"蒙对一题但整体理解为零"的尴尬。
  - **3300 人工时 + 50 人 QA**的数据质量在 benchmark 里算顶配。
  - **但**：(1) 800 视频 3200 题的规模撑不起"旗舰标准"； (2) 非线性评分函数完全是 heuristic——$(N/4)^2$ 为什么不是其他指数？没有理论支撑；(3) 全部是多选题，开放问答能力完全没测到。
- **Rating**: `2` 👀 ，评分策略有参考价值但本质还是"换个更难考卷"的 benchmark 老套路

### Attention Sink Survey
- **Title**: Attention Sink in Transformers: A Survey on Utilization, Interpretation, and Mitigation
- **Institutes**: Tsinghua University, Meituan LongCat Team, HKU, Xiamen University, UMich, Ohio State
- **Source**: [link](https://arxiv.org/abs/2604.10098) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-AttentionSink]]
- **核心**: 第一个系统梳理 Attention Sink 现象的 survey。AS 指 Transformer 中少量无信息 token 吸引不成比例的高注意力权重。从三个维度组织：Utilization（保护或重分配 sink token）；Interpretation（softmax 约束、异常值回路、位置偏置、几何锚点四大理论）；Mitigation（门控注意力、改 softmax、可学习 bias）。
- **锐评**: 
  - **20 个作者的 survey**——这本身就是一个 red flag。综述不是靠人多堆出来的，靠的是洞察。
  - **三个维度与其实按"你对 AS 的态度"在分桶**——分类本身没什么信息量。
  - **Mitigation 部分列了一堆方法，但没有任何 benchmark 比较**。作为 survey，连一张"各方法在相同任务上孰优孰劣"的总结表都没有的话，就只是 glorified bibliography。
- **Rating**: `2` 👀 ，作为 entry-level reading guide 可用，但缺乏 cross-method comparison 和理论层面的 synthesis

### OmniShow HOI
- **Title**: OmniShow: Unifying Multimodal Conditions for Human-Object Interaction Video Generation
- **Institutes**: CUHK 等
- **Source**: [link](https://arxiv.org/abs/2604.11804) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-OmniShow]]
- **核心**: 面向 HOI Video Generation，同时接受文本、参考图像、音频、姿态四种条件输入生成视频。方法：(1) Unified Channel-wise Conditioning；(2) Gated Local-Context Attention；(3) Decoupled-Then-Joint Training。自建 HOIVG-Bench 基准。
- **锐评**: 
  - **"Omni"前缀是 2026 年视频生成领域的通胀信号**——把能加的条件全堆上去不等于 unified。
  - Unified Channel-wise Conditioning 大概率是 channel-wise concatenation 的 rebranding。
  - Gated Local-Context Attention 局部注意力 + 门控在 audio-driven video generation 中已有大量先例。
  - **自建 benchmark 然后在自己模型上刷 SOTA，缺乏第三方验证容易变成 self-serving evaluation**。
- **Rating**: `2` 👀 ，多模态统一条件生成的方向是对的，但方法层面的贡献需要全文验证

### DiPO
- **Title**: DiPO: Disentangled Perplexity Policy Optimization for Fine-grained Exploration-Exploitation Trade-Off
- **Institutes**: 待确认
- **Source**: [link](https://arxiv.org/abs/2604.13902) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-DiPO]]
- **核心**: RLVR 训练中存在 exploration-exploitation trade-off 管理问题。提出用 perplexity 将样本空间解耦为 exploration（高 perplexity）和 exploitation（低 perplexity）两个子空间，配合 bidirectional reward allocation 机制。在 mathematical reasoning 和 function calling 上验证。
- **锐评**: 
  - **perplexity 和 reasoning difficulty 之间的关系本身就是个未经检验的假设**。一个多步数学题的答案格式如果是 model 熟悉的，perplexity 可以很低，但 reasoning chain 可以完全错误。
  - "bidirectional reward allocation"本质上就是 perplexity-conditioned reward reshaping。关键在于 reshaping 是否保持了最优策略不变。
- **Rating**: `2` 👀 ，idea 方向对，但 perplexity 这个 proxy 太粗糙

### Externalization
- **Title**: Externalization in LLM Agents: A Unified Review of Memory, Skills, Procedures and World Models
- **Institutes**: Shanghai Jiao Tong University, Sun Yat-sen University, Shanghai AI Lab, Carnegie Mellon University, OPPO
- **Source**: [link](https://arxiv.org/abs/2604.08224) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-Externalization]]
- **核心**: 提出"Externalization"作为统一框架，解释 LLM Agent 的能力演进路径：从模型内部权重 → 上下文 → 外部运行时基础设施。将 Memory、Skills、Protocols、Harness 四大组件分别对应"认知外包"的不同维度，借用 Donald Norman 的 cognitive artifacts 理论做概念包装。54 页，纯综述。
- **锐评**: 
  - **本质上做的是对已有工程实践的重新命名和分类**。RAG 叫"Memory Externalization"，tool-use 叫"Skill Externalization"——换个马甲不等于发现了新原理。
  - "Harness Engineering"说的就是 LangChain、AutoGen、CrewAI 干的事情。**把工业界已经做完的事情起个学术名字再发回来，这是典型的 circular innovation**。
  - **54 页对于一个没有 falsifiable claim 的纯概念框架来说，信息密度值得怀疑**。
  - 没有 Ablation、没有 benchmark、没有 comparative analysis——"Unified"体现在哪里？
- **Rating**: `2` 👀 ，框架整理有参考价值，但 54 页讲一句话的事

### HY-Embodied-0.5
- **Title**: HY-Embodied-0.5: Embodied Foundation Models for Real-World Agents
- **Institutes**: Tencent Robotics X, HY Vision Team
- **Source**: [link](https://arxiv.org/abs/2604.07430) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-HYEmbodied]]
- **核心**: 腾讯混元团队提出 embodied foundation model 系列（2B 和 32B），用 Mixture-of-Transformers (MoT) 架构做模态专用计算 + latent tokens 增强感知，用自进化 post-training 提升推理能力，再用 on-policy distillation 把 32B 的能力压到 2B 上。22 个 benchmark 上测试，2B 在 16 个上超过同规模 SOTA。
- **锐评**: 
  - **"0.5" 命名暴露了一切**——这明显是 Hunyuan 系列的 incremental update。
  - **MoT + latent tokens 不是什么新东西**，Mixture-of-Experts 和 modality-specific routing 已经是 2024-2025 的标准配方了。
  - **"22 个 benchmark" 听起来唬人**，但需要警惕 benchmark shopping。
  - **32B "comparable to Gemini 3.0 Pro"** 这个 claim 值得打问号。
- **Rating**: `2` 👀 ，腾讯的工程实力毋庸置疑，但从学术贡献角度，这套组合拳缺乏核心 insight

### GenerativeWorldRenderer
- **Title**: Generative World Renderer
- **Institutes**: Alaya Studio, National Taiwan University, The University of Tokyo
- **Source**: [link](https://arxiv.org/abs/2604.02329) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-GenerativeWorldRenderer]]
- **核心**: 现有 synthetic dataset 太假，inverse rendering 泛化不了。从 Cyberpunk 2077 和黑神话里用 ReShade 截 G-buffer，凑了 4M 帧 RGB+5 通道 G-buffer 数据集。拿这个微调 DiffusionRenderer。又提了 VLM-based evaluation 替代人工评测。
- **锐评**: 
  - **标题叫 "Generative World Renderer"，实际是个 dataset paper**。核心方法 = fine-tune DiffusionRenderer。
  - 4M 帧、长序列、多天气采集工程活干得扎实，dataset 本身有价值。
  - 但法线从 depth 有限差分重建，在 depth discontinuity 和遮挡处必然炸。这个问题被轻描淡写地跳过了。
  - Gated access + 商业游戏素材，数据集能活多久取决于游戏厂商的心情。
- **Rating**: `2` 👀 ，dataset 工程值得尊重，但 method 零创新

### CARLA-Air
- **Title**: CARLA-Air: Fly Drones Inside a CARLA World -- A Unified Infrastructure for Air-Ground Embodied Intelligence
- **Institutes**: The University of Hong Kong
- **Source**: [link](https://arxiv.org/abs/2603.28032) 📰 HF Daily **📒 论文笔记**: [[Papers/2603-CARLAAir]]
- **核心**: 把 CARLA（地面驾驶）和 AirSim（无人机）塞进同一个 UE4 进程，通过继承+组合解决 UE4 单 GameMode 冲突，仅改 35 行上游代码就实现了空地联合仿真。保留两套原生 API，共享物理 tick 和渲染管线。5 个 workload 验证，~20 FPS、3 小时 357 次 reset 零崩溃。
- **锐评**: 
  - **这是一篇工程集成论文，算法创新为零**——核心贡献就是 ~1405 行 C++ 插件代码。
  - 但缝得确实漂亮：UE4 GameMode 冲突用"继承 CARLA + 组合 AirSim actor"解决，只动 2 个上游文件。
  - **~20 FPS 的吞吐量自称支持 RL 训练，实际上只能做 policy evaluation**。
  - 基于 UE4，UE5 已经是行业标准，这个技术债迟早要还。
- **Rating**: `2` 👀 ，工程质量和集成设计值得肯定，但零算法创新

### EXAONE 4.5
- **Title**: EXAONE 4.5 Technical Report
- **Institutes**: LG AI Research
- **Source**: [link](https://arxiv.org/abs/2604.08644) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-EXAONE45]]
- **核心**: LG 把 EXAONE 4.0 32B 语言模型拼上一个从零训练的 1.2B vision encoder，做成 33B dense VLM。数据策略偏向 document/OCR，post-training 走 DPO + GRPO 标准流程。在 document understanding 和 Korean reasoning 上表现不错。
- **锐评**: 
  - **BabyVision 18.8**——比 Qwen3.5-27B 的 44.6 低了整整 26 分，说明模型连最基本的物体识别都成问题。
  - HallusionBench 63.7 同样拉胯。暴露了经典问题：document-centric 数据过拟合，模型学会了读 PDF 但连"图里有几只猫"都答不对。
  - **整篇 report 没有任何有意义的 ablation**：vision encoder 为什么选 1.2B？GRPO reward 怎么设计的？一概没有。
- **Rating**: `2` 👀 ，LG 的工业 document pipeline 有工程参考价值，但学术贡献几乎为零

### OpenWorldLib
- **Title**: OpenWorldLib: A Unified Codebase and Definition of Advanced World Models
- **Institutes**: DataFlow, NUS
- **Source**: [link](https://arxiv.org/abs/2604.04707) 📰 HF Daily **📒 论文笔记**: [[Papers/2604-OpenWorldLib]]
- **核心**: 给 world model 下了个定义——"以 perception 为中心，带 interaction 和 long-term memory"，然后做一个统一推理框架，把各种 world model 模型集成到一起。42 位作者，28 页。
- **锐评**: 
  - 这就是一篇标准的 **"library paper"**——代码写得不错值得发篇 arxiv，但离学术贡献差得远。
  - 那个所谓的 "clear definition" 本质上是 "perception + interaction + memory" 三个词的排列组合，任何一个做 world model 的人脑子里早就有类似概念了。
  - **42 个作者做一个 inference wrapper，人均工作量约等于写一个 `import` 语句**。
- **Rating**: `1` 💤 ，代码库可以 star，论文不必 cite

## 月度洞察

### Pattern 发现

1. **World Model 赛道：incremental engineering > novel insight**
   - HybridMemory、MultiWorld、GenerativeWorldRenderer 都是工程整合，缺少真正改变问题 formulation 的核心 idea
   - 唯一亮点是 SpatialEvo 的 DGE 设计——用确定性几何替代 model voting，但适用场景极窄

2. **Agentic RL：failure mode 诊断有价值，但解决方法偏弱**
   - RAGEN-2 的 template collapse 发现很准，但 SNR-Aware Filtering 只是 heuristic
   - Credit Assignment 赛道（SOLAR-RL、GiGPO、ADMIRE）已覆盖大部分设计空间

3. **VLA/VLM：工程整合为主，缺少架构突破**
   - HiVLA hierarchical 设计合理但 baseline 选择性缺席
   - HY-Embodied-0.5、EXAONE 4.5 都是"大厂工程报告"类型

4. **Benchmark 赛道：从"换考卷"到"换评分方式"**
   - Video-MME-v2 的非线性评分有参考价值
   - Claw-Eval 的 trajectory-aware grading 和 44% miss rate 是有价值的数据点

### 下一步关注

- **SpatialEvo 的 DGE 范式**能否扩展到更多场景？
- **RAGEN-2 template collapse** 是否有更 principled 的解决方案？
- **Agent-World 的 environment scaling** 真实场景效果如何？
