### ARIS
- **Title**: ARIS: Autonomous Research via Adversarial Multi-Agent Collaboration
- **Institutes**: (作者未在 arXiv 页面标注机构)
- **Source**: 📰 HF Daily ⬆️ 81   **📒 论文笔记**: [[Papers/2605-ARIS]]
- **核心**: 做自主科研系统的开源框架。核心卖点是"跨模型对抗协作"——executor 推进任务，另一家模型的 reviewer 独立审计、打分、要求修订。设计了三层架构：Execution（65+ skills）、Orchestration（5 个 workflow）、Assurance（三阶段证据审计）。提出了 5 条设计原则，工程量不小。
- **锐评**: 
  - **问题意识正确但验证为零**。"单 agent 长期任务不可靠"这个判断我认同，cross-model adversarial 也是合理的解法。但全文没有一个受控实验！全是"部署了 65+ skills""跑了 8 小时"这种观察性证据。cross-model 真的比 same-model self-critique 好吗？catch rate 高多少？false positive 呢？——全没数据。
  - **Assurance Layer 设计精密但谁来审计审计者？** 三阶段审计cascade 听起来很严谨，但如果 executor 和 reviewer 都有盲区呢？系统只是把信任从 executor 转移到 reviewer，根本问题没解决。
  - **安全问题未解**：repository-level review 会把整个代码库发给外部 API。做私有研究？没门。
  - **和外竞品对比浮于表面**：Table 4 列了 feature checklist，但没有 head-to-head 的性能对比。你比 AI Scientist 好在哪？快？准？省？——不知道。
  - **工程量值得肯定**：65+ skills、6 个 MCP bridges、完整审计 pipeline，开源可用。这是实打实的贡献。
  - **诚实加分**：明确承认"无受控评估、correctness 无保证、安全问题未解"。比很多 paper 强。
- **Rating**: `3` — 系统设计有洞察，工程扎实，但核心 hypothesis 零验证。Appendix E 的 benchmark 协议写得很好，就是没跑。等有数据再说话。

### DTap
- **Title**: DecodingTrust-Agent Platform (DTap): A Controllable and Interactive Red-Teaming Platform for AI Agents
- **Institutes**: UIUC, Stanford, UC Berkeley, UW-Madison, UChicago, CMU
- **Source**: 📄 arXiv   **📒 论文笔记**: [[Papers/2605-DTap]]
- **核心**: DTap 是首个 AI Agent 红队测试平台，覆盖 14 个领域、50+ 模拟环境（Google Workspace/PayPal/Slack），提供 DTap-Red 自动化红队方法和 DTap-Bench 数据集（带 verifiable judges）。
- **锐评**: 
  - **作者阵容豪华**：Percy Liang + Dawn Song + Bo Li，这是 AI 安全圈的"复仇者联盟"，基本意味着这篇 paper 不会太差。
  - **但 platform paper 的通病**：279 页、148 图——这更像是一个"工程报告"而非研究论文。核心 novelty 是平台本身还是 DTap-Red 的红队方法？从 abstract 看不太清晰。
  - **关键问题未回答**：DTap-Red 的攻击成功率如何？verifiable judge 的准确率是多少？baseline 是什么？没有这些数字，"首个"这个 claim 的含金量存疑。
  - **模拟环境的永恒难题**：模拟的 PayPal/Slack 能多接近真实系统？攻击在模拟环境成功不代表真实环境也成功——这 gap 有多大？论文需要严肃讨论。
  - **与 WebArena 的关系**：WebArena 也是模拟环境 benchmark，DTap 的 differentiation 是红队测试。但红队测试本身是个方法问题，不是平台问题。平台只是载体，方法才是灵魂。
- **Rating**: `3` — 平台工作，工程量大但 research novelty 待验证；安全方向的 agent 研究者值得一看，其他人可等具体实验数据出来再判断。

### AgentTrust
- **Title**: AgentTrust: Runtime Safety Evaluation and Interception for AI Agent Tool Use
- **Institutes**: Independent Researcher (Chenglin Yang)
- **Source**: 📄 arXiv   **📒 论文笔记**: [[Papers/2605-AgentTrust]]
- **核心**: 针对 AI Agent 工具调用的运行时安全拦截框架，在 Agent 和工具之间插入拦截层，通过 shell 去混淆、170 条规则、多步攻击链检测、SafeFix 建议和缓存感知的 LLM Judge，在生产环境中实时返回 allow/warn/block/review 判决，300 场景内部基准 95.0% 判决准确率，630 场景独立基准 96.7%。
- **锐评**: 这篇论文工程完整度很高——5000 行核心代码、192 单元测试、四个故障安全不变量、AGPL-3.0 开源，但方法论创新有限，本质上是一个精心设计的规则引擎 + 一些启发式组件。**硬伤**：(1) baseline 对比不公平——B1 只有 50 个 pattern vs AgentTrust 170+ 条规则，应该对比相同规则数量；(2) 混合模式（rules + LLM Judge）准确率反而下降（88% vs 95%），LLM Judge 的价值存疑；(3) 630 场景基准是"后补丁"结果，不是 zero-shot，有 overfit 嫌疑。**亮点数字**：shell 混淆 payload ~93% 准确率（这确实是个实用痛点），0.3ms 延迟（工程务实）。**暴露问题**：RiskChain 在当前基准上无 measurable 效果——多步攻击链检测的价值在哪里？ShellNormalizer 纯文本设计故意不执行 eval，存在绕过空间。作者自承七项局限（L1-L7）诚实度可嘉，但也暴露方法的本质局限：静态分析天花板。
- **Rating**: `3`，工程扎实的工业级方案，但学术创新有限，适合作为基础设施但不应过度依赖。

### OpenSearch-VL
- **Title**: OpenSearch-VL: An Open Recipe for Frontier Multimodal Search Agents
- **Institutes**: 未标注（arXiv 页面未列出作者机构）
- **Source**: 📄 arXiv   **📒 论文笔记**: [[Papers/2605-OpenSearchVL]]
- **核心**: 首个完全开源的 Multimodal Deep Search Agent 配方——数据管线（Wikipedia 路径采样+模糊实体重写+源锚点视觉定位）、工具环境（检索+增强+OCR）、训练算法（Fatal-Aware GRPO：fatal step masking + one-sided advantage clamping 处理 cascading tool failures）。7 个 benchmark 平均 +10 分，32B 模型超越 Gemini-2.5-Pro。
- **锐评**: 方法设计有诚意，但"开源配方依赖闭源评估"是个笑话——reward 的 accuracy 用 GPT-4o judge、process quality 用 GPT-5.4 judge，训练成本是有了，评估成本谁买单？更讽刺的是 judge 只评估文本 query，视觉操作的质量完全不在奖励信号里——一个"multimodal search agent"的 RL 训练竟然对视觉操作视而不见。数据管线的消融做得扎实（-11.5/-10.3/-8.2/-1.3），source-anchor grounding 的设计确实精巧（强制多跳推理而非 shortcut retrieval），但 fatal-aware GRPO 与 SOLAR-RL 的 first failure point detection 本质是同一思路，novelty 有限。最亮眼的数字是 one-sided clamping 相比 hard masking 提升 4.1 分——这才是真正的 contribution，但全文只有一处曲线图展示，可惜。机构信息缺失是什么情况？开源精神连机构都不敢标？
- **Rating**: `3` —— 工程完整、开源有价值，但核心方法 novelty 有限且评估依赖闭源模型形成悖论，适合作为 agentic RL baseline 参考，不值得追 deep dive。

### Rollout Pass-Rate Control
- **Title**: Rollout Pass-Rate Control: Steering Binary-Reward RL Toward Its Most Informative Regime
- **Institutes**: 未明确（11 位作者，从姓名推测为中国团队，但论文与 arXiv 页面均未标注机构）
- **Source**: 📄 arXiv   **📒 论文笔记**: [[Papers/2605-RolloutPassRateControl]]
- **核心**: Binary reward RL 训练中，pass rate 极端偏斜（0/8 或 8/8）的 rollout groups 完全没有 contrastive signal。论文证明 50% pass rate 是最优工作点，提出 Prefix Sampling：对 hard groups replay 成功 prefix 作"助攻"，对 easy groups replay 失败 prefix 作"障碍"。Qwen3-14B 上实现 2.01x 端到端加速，SWE-bench Verified 从 0.247 提到 0.295。
- **锐评**: 
  - **方法设计聪明**：双向 steering 的洞察很实在——只救 hard tasks 是多数人的直觉，但主动 handicap easy tasks 来制造 contrastive signal 这个想法有新意。Prefix masking 解耦状态操纵和 credit assignment 的设计也很干净。
  - **证据扎实但范围窄**：ablation 清晰分解了 replay（1.61x）、adaptive control、bidirectional coverage 三个贡献，没有糊弄。但全部实验只验证了 N=8 group size，对其他 N、continuous reward 场景的泛化性完全没提——binary reward + grouped rollouts 的前提条件是否太苛刻？
  - **机构信息缺失很业余**：11 人作者团队，机构一栏空白。arXiv 页面没有，论文正文也没写。这不是小问题——影响可复现性判断，也让人怀疑是不是有什么不便公开的。
  - **Valid groups 增加不能完全解释加速**：PS-fix 保留 85.1 个 valid groups 但收敛比 PS-ada（80.2 groups）更慢。说明 signal 质量（落在 50% 区间）比 signal 数量更重要，这个 point 论文论述到位。
  - **50% target 敏感性未探索**：deadzone δ=0.03、step size η=0.05 都是拍脑袋选的，没有 sensitivity sweep。理论上 50% 是 entropy 最大点，但实践中 40%-60% 是否都 work？论文没说。
- **Rating**: `3`——方法简洁有效，问题定位精准，但适用范围有限、机构信息缺失、超参数敏感性分析不足。有参考价值，但不是"必读"级别。

### PRISM
- **Title**: Beyond SFT-to-RL: Pre-alignment via Black-Box On-Policy Distillation for Multimodal RL
- **Institutes**: HKUST (Guangzhou), Tsinghua, NTU, RUC, USTC, UCAS
- **Source**: 📰 HF Daily ⬆️ 39   **📒 论文笔记**: [[Papers/2605-PRISM]]
- **核心**: 指出 SFT 的 token-level 均匀模仿在多模态场景导致"分布漂移"——视觉 grounding 和逻辑推理以不同方式退化。解决方案是在 SFT 和 RLVR 之间插入 MoE discriminator 做对抗式 on-policy distillation，双专家分别纠正感知和推理漂移。Qwen3-VL 4B/8B 上比标准 SFT→RLVR 分别 +4.4/+6.0。
- **锐评**: 问题诊断是真正贡献——"SFT 让模型既不匹配示范分布也不保留原始分布"这个观察很敏锐，多模态漂移异质性的洞察也有价值。但方法本身不够优雅：用四个 2B 模型做 MoE discriminator，训练复杂度上去了，对比 dense 方案的计算开销却没报？6K 样本筛到 2K 用于 RLVR，这个筛选逻辑的泛化性存疑。更关键的是，只测了 Qwen3-VL——如果你声称这是"通用范式"，至少在 LLaVA、InternVL 上跑一跑？还有依赖 Gemini 3 Flash 生成示范，如果 supervision 本身有系统性 bias，你的 alignment 在纠正什么？Ablation 里 w/o SFT 的 -16.8 很说明问题——这个三阶段流水线缺一不可，但也意味着三个阶段都得调好，工程门槛不低。最后，移除 KL 正则化是个冒险决定，虽然声称会"阻碍纠正漂移"，但没有消融这个选择的风险——有没有 mode collapse？生成多样性下降多少？
- **Rating**: `2` — 问题诊断有价值，方法设计有想法，但验证不够充分，工程复杂度高，泛化性未证明。了解即可。

### Pixels to Tokens
- **Title**: From Pixels to Tokens: A Systematic Study of Latent Action Supervision for Vision-Language-Action Models
- **Institutes**: 未从网页抓取到机构信息
- **Source**: 📄 arXiv   **📒 论文笔记**: [[Papers/2605-PixelsToTokens]]
- **核心**: 系统性对比 VLA 模型中 latent action supervision 的两种视角（image-based 用于轨迹正则化、action-based 用于目标空间统一）及四种集成策略，发现"直接让 VLM 预测离散 latent tokens"是最简单也最有效的方式。Image-based 擅长长程任务，action-based 擅长运动复杂任务。
- **锐评**: 这篇论文的定位非常精准——不追求 novelty bombing，而是做了一件事：把前人碎片化的 latent action 集成方式统一到一个基线下做消融。结论简洁有力：不需要花里胡哨的 conditional decoding 或 implicit alignment，直接预测 latent tokens 就够了。实验扎实，LIBERO、RoboTwin 2.0、真实机器人三管齐下，OOD 场景 LA-Cond 从 17 提到 70 这个数字很亮眼。但硬伤也有：作者名单 7 人，真实机器人实验却只有单臂平台，跨形态泛化的 claim 站不住脚；机构信息未标注，显得不够严谨；image-based 和 action-based 的分野虽然定性解释了，但缺 ablation——为什么 FFT+1D conv 就是 action-based 的最优设计？选这个架构的理由是什么？另外，latent action model 本身的质量对最终效果的影响没有充分分析，只说"open question"。整体来说，是一篇"把显而易见的事做对"的工程论文，价值在于为后续研究提供了清晰的 baseline 和实验范式，但创新性有限，更多是收拾前人留下的烂摊子。
- **Rating**: `3` —— 工程扎实但创新有限，为 VLA latent action 研究提供了有价值的 baseline 和实验范式，但没有突破性的 insight。

### ConsisVLA-4D
- **Title**: ConsisVLA-4D: Advancing Spatiotemporal Consistency in Efficient 3D-Perception and 4D-Reasoning for Robotic Manipulation
- **Institutes**: 未明确标注（代码仓库：iLearn-Lab）
- **Source**: 📄 arXiv (CVPR 2026 Accepted)   **📒 论文笔记**: [[Papers/2605-ConsisVLA4D]]
- **核心**: 通过三级一致性（跨视角语义、跨物体几何、跨场景时空）将 VLA 从 2D 视觉推理升级到 3D 感知+4D 推理。用 FiLM+Top-K 显式筛选、余弦衰减融合、训练显式监督→推理隐式知识三招，在 LIBERO 达到 98.1%（+21.6% vs OpenVLA），同时 FLOPs 减半、推理加速 2.3×。
- **锐评**: 
  亮点是真实的：LIBERO 98.1% 确实 sota，效率提升"加功能反而更快"的设计有说服力，余弦衰减 α_l 的消融数据漂亮（98.1% vs 94.4%/95.9%）。但问题也很明显——**多视角依赖是硬伤**，CV-Aligner 明确需要至少双视角，单目场景直接不适用，这比"需要深度传感器"的限制更窄。实验设计里 Real-world 全是双臂/多摄像头任务，刻意回避了单目问题。**"4D 推理"的宣称过于宏大**，CS-Thinker 只学习"固定视角 i* 的动态预测"，根本没验证跨视角泛化，本质还是单视角时序预测，套个"4D"帽子有 overclaim 嫌疑。数据成本方面，VGGT 和 CoTracker 监督需要深度/跟踪标注，文章只吹效率不谈标注开销，避重就轻。LIBERO Long（95.6%）比 Spatial/Object 低 3+ 个点，长程性能下降但没深入分析，real-world 叠衣任务成功率直接不报具体数字，选择性透明。对比 SpatialVLA 的 baseline 选择有公平性可言，但核心问题——**这些"一致性"是否真的需要这么复杂的架构？** Top-K=32 没消融，双臂 chunk=25 vs 单臂 chunk=8 的时序对齐没解释，感觉是模块堆叠而非最小必要设计。CVPR 2026 收了说明领域缺这类工作，但"三级一致性"的故事性>必要性。
- **Rating**: `3` —— 方法有创新（训练显式→推理隐式的设计聪明），实验扎实，但对单目场景不适用限制了泛用性，"4D 推理"宣称存在 overclaim，长程任务性能衰减暴露泛化问题。值得精读，但别被一致性框架的复杂性迷惑。
