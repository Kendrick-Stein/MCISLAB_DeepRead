## 论文点评

### AutoFocus
- **Title**: AutoFocus: Uncertainty-Aware Active Visual Search for GUI Grounding
- **Institutes**: Wuhan University of Technology; Institute of Automation, Chinese Academy of Sciences
- **Source**: [link](https://arxiv.org/abs/2605.02630v1) 📄 arXiv  **📒 论文笔记**: [[Papers/2605-AutoFocus]]
- **核心**: 高分辨率 GUI 下 VLM grounding 因分辨率 mismatch 严重退化。AutoFocus 发现 token-level perplexity 与 grounding 正确率高度相关，于是把 perplexity 转化为各向异性高斯空间概率场来指导 zoom-in。核心流程：多采样坐标 → axial perplexity 建模方向不确定性 → Gaussian splatting 生成概率场 → global-local region proposals → Shape-Aware Zooming → 视觉聚合选最优。Training-free，即插即用。SS-Pro 上 UI-Venus-7B 从 50.3% 拉到 67.8%（+17.5），Qwen2.5-VL-72B 从 47.8% 拉到 65.1%（+17.3），增益集中在 icon grounding 和高密度布局场景。
- **锐评**: Core insight 不错，perplexity 作为不确定性信号比 attention map 或 gradient saliency 更 elegant，SS-Pro 上的提升也确实 impressive。但硬伤有三：(1) **计算开销完全回避**——N=5 采样就是 5 次 full forward pass，加上验证、各 region proposal 的 zoom-in、聚合，总推理成本可能是 baseline 的 10-20x，论文对此只字不提，这在 "training-free" 的 claim 下是严重的信息缺失；(2) **Gaussian machinery 收益有限**——ablation 显示全框架比"仅用 local proposals"在 SS-Pro 上只高 2.3%，大部分增益来自 zoom-in 本身，那些精巧的各向异性 splatting、axial PPL 等设计边际贡献很小（+1.3%），方法论上的 novelty 被自身 ablation 打了折扣；(3) **β 需要 per-model 调参**（GTA1 用 80，其他用 50），"training-free" 在实践中变成了"需要 per-model hyperparameter search"。此外，验证模块用同家族模型做 self-confirmation 有系统偏差风险，SS-V2 上增益有限（+1.2%~+4.9%）说明方法主要吃 hard case 的红利，且没有对比 random cropping 或 uniform grid 这些 trivial baseline。一个聪明的 idea，但包装得比实际更 fancy。
- **Rating**: `3` 🔥 有参考价值的 training-free 细化框架，perplexity-uncertainty 的 insight 可推广，但计算开销未量化、复杂设计边际收益有限，实际部署价值存疑

### AcademiClaw
- **Title**: AcademiClaw: When Students Set Challenges for AI Agents
- **Institutes**: Shanghai Jiao Tong University, SII (Shanghai Institute of Innovation), GAIR (General Artificial Intelligence Research)
- **Source**: [link](https://arxiv.org/abs/2605.02661v1) 📄 arXiv  **📒 论文笔记**: [[Papers/2605-AcademiClaw]]
- **核心**: OpenClaw 生态首个学术级 benchmark，80 个来自大学生真实学术流程的复杂任务（230→80 筛选），覆盖 25+ 领域含 16 个 GPU 任务。最强模型 Claude Opus 4.6 仅 55% 通过率，但真正有价值的发现是：token 消耗与输出质量几乎零相关（r=-0.03），以及三种行为表型（read-first/execute-first/minimalist）的效率-安全差异。
- **锐评**: 任务收集方法有新意（学生贡献 + 专家筛选），但 80 个任务分到 25+ 领域，每个领域平均 3 个样本，统计上根本撑不起 domain-level 结论。单次尝试无 Pass@k，无法区分随机失败和系统性失败。论文声称 "first agent benchmark whose tasks originate entirely from university students"，但 SWE-bench 的 GitHub issues 也来自真实用户，这个 "first" 站不住。模型版本号（GPT-5.4、Claude Opus 4.6）指向未来模型，可复现性存疑。Gemini 消耗 5.4× token 得分反而最低的发现有趣，但 exec:read = 28:1 的行为表型是否是 Gemini 3.1 Pro 的特例而非 execute-first 策略的通病？需要更多模型验证。安全审计中 S3（boundary compliance）53 分差距亮眼，但 Gemini 的低分是否因为 exec-first 策略增加了违规面积，还是 Gemini 本身的安全对齐问题？论文没做控制实验。LLM-as-Judge 用 GPT-5.2 而非独立 judge，同一厂商模型家族的 bias 没有讨论。
- **Rating**: `2` 👀 任务来源方法有新意，GPU 任务填补空白，但 80 任务规模和单次评估设计限制了结论的可靠性，属于 "有参考价值但别太当真" 的 benchmark 论文

### SFI-Bench
- **Title**: From Where Things Are to What They Are For: Benchmarking Spatial-Functional Intelligence in Multimodal LLMs
- **Institutes**: Mila - Québec AI Institute, UdeM / NYU / Apple / Canada CIFAR AI Chair
- **Source**: [link](https://arxiv.org/abs/2605.02130v1) 📄 arXiv  **📒 论文笔记**: [[Papers/2605-SFIBench]]
- **核心**: 提出 SFI-Bench，从 cognitive spatial reasoning（conditional counting、path reasoning、layout inference）和 functional reasoning（affordance、operation planning、troubleshooting）两个维度评估 MLLM。1555 题 MCQ，134 个 egocentric indoor video。最强模型 Gemini-3.1-Pro 仅 73.8%，conditional counting 是最大瓶颈（59.1%）。RLVR 训练的 reasoning 模型相比 instruct 版本几乎没有增益，模型不依赖时序连续性，visual input 不可被 caption 替代。
- **锐评**: 任务设计有亮点——将 counting 重新包装为 compositional logical reasoning、将 troubleshooting 与 web search 绑定，比 VSI-Bench 之类纯 perception benchmark 更接近 agentic 场景。但硬伤也很明显：1555 题、134 个视频，规模太小，统计置信度撑不起"benchmark"这个词；纯 MCQ 格式回避了开放式评估，option 质量直接决定难度；所谓的"functional reasoning"实质是 retrieval + tool-use，而非真正的 affordance understanding——operation planning 查手册、troubleshooting 搜网页，测的是搜索引擎用得好不好，不是模型懂不懂物体功能。数据构建全程依赖 Gemini-2.5-Pro 提取 metadata 再模板生成问题，引入了系统性 bias。帧序打乱实验只用 200 个样本就下"模型不依赖时序"的结论，样本量太薄。最亮眼的发现是 RLVR 在 spatial-functional 任务上完全无效（Finding 3），这暗示当前 RLVR 训练目标和 spatial cognition 之间可能存在 distribution mismatch，值得深挖。GPT-5 的 web search 带来 +8% 增益但低推理预算下反降，说明 tool use 需要强推理能力做前提，这个发现有 practical value。
- **Rating**: `2` 👀 任务设计有新意但规模太小、MCQ-only、"functional reasoning"实质偏窄，作为 benchmark 的长期影响力有限
