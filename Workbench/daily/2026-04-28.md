## 论文点评

### Agentic World Modeling
- **Title**: Agentic World Modeling: Foundations, Capabilities, Laws, and Beyond
- **Institutes**: NTU, NUS, CUHK, Oxford 等（40+ 作者大联盟）
- **Source**: [link](https://arxiv.org/abs/2604.22748) 📰 HF Daily ⬆️ 210   **📒 论文笔记**: [[2604-AgenticWorldModel]]
- **核心**: "levels x laws" 框架——L1 Predictor → L2 Simulator → L3 Evolver 横跨 physical/digital/social/scientific 四域。400+ 论文综述，统一 MBRL、视频生成、GUI Agent、Multi-agent、AI Scientist。主张：World Model 终极形态是能自主修正的 Evolver。
- **锐评**: Taxonomy 设计精巧，能覆盖现有工作核心差异。但 Survey 通病——taxonomy 本身不等于 insight。L3 Evolver 把"模型自主修正"当终点几乎在说 AGI 是必然形态，claim 远超 evidence。40+ 作者像"朋友圈论文"。真正价值在于 failure mode 系统化，需全文判断。
- **Rating**: `2` 👀 Taxonomy 工整但缺少 surprise，L3 过于理想化。

### Skills to Talent
- **Title**: From Skills to Talent: Organising Heterogeneous Agents as a Real-World Company
- **Institutes**: 未标注
- **Source**: [link](https://arxiv.org/abs/2604.22446) 📰 HF Daily ⬆️ 108   **📒 论文笔记**: [[2604-SkillsToTalent]]
- **核心**: Multi-agent 从"固定团队+硬编码协调"升级到"企业级组织架构"。Talents 封装 agent 身份、Talent Market 按需招募、E²R tree search 形成自组织闭环。PRDBench 84.67%，+15.48pp SOTA。
- **锐评**: 问题定义比方法锋利——"缺组织架构层"诊断高明。企业 HR/绩效考核隐喻有趣。但 Talent Market 机制未讲清——如何防垃圾 agent？质量保证？+15.48pp SOTA baseline 未说明。E²R vs HTN vs hierarchical RL 是否只是换词包装？termination guarantee 在 open-ended tasks 是悖论。
- **Rating**: `2` 👀 概念有趣但细节模糊，"组织架构层"idea值得 follow，但 evidence 不足以支撑宏大 claim。

### VLA Safety Survey
- **Title**: Vision-Language-Action Safety: Threats, Challenges, Evaluations, and Mechanisms
- **Institutes**: National University of Singapore, Monash University, Peking University
- **Source**: [link](https://arxiv.org/abs/2604.23775) 📰 arXiv   **📒 论文笔记**: [[2604-VLASafety]]
- **核心**: 首个 VLA 安全 Survey，双 timing 轴框架（attack timing: training/inference + defense timing: training/inference）配对威胁与防御。覆盖 training-time threats（data poisoning、backdoors）、inference-time attacks（semantic jailbreaks、perturbations）及对应防御。六大部署场景 cross-domain patterns（sim-to-real gap、safety-capability Pareto）。
- **锐评**: Survey 无新方法，评价标准是"整理系统/框架清晰/缺口暴露"——三项都达标。双 timing 轴解决 VLA safety 文献碎片化，把 robot learning + adversarial ML + AI alignment 三路缝合。六大部署场景 cross-domain patterns 有深度。但威胁描述偏薄（proprioceptive state-space poisoning、freezing attacks 点到为止）。benchmarks 部分像 catalogue 缺少批判性对比。"路线图式" survey——给新人提供坐标系，给研究者定位归属，不指望学到技术细节。
- **Rating**: `2` 👀 Survey 性文献整理，框架清晰但无新方法，适合入门了解领域全貌。

### M²-VLA
- **Title**: $M^2$-VLA: Boosting Vision-Language Models for Generalizable Manipulation via Layer Mixture and Meta-Skills
- **Institutes**: 未明确
- **Source**: [link](http://arxiv.org/abs/2604.24182v1) 📄 arXiv   **📒 论文笔记**: [[2604-M2VLA]]
- **核心**: Mixture of Layers (MoL) 从 VLM 多层选择性提取 + Meta Skill Module (MSM) 引入强归纳偏置。声称不损害 VLM 泛化能力，实现 zero-shot manipulation。
- **锐评**: 笔记基于 abstract，核心数字全无。MoL/MSM 具体设计、训练开销未知。tag 错误——`world-model` 不适用。与同期 LibraVLA (97.2% LIBERO)、CF-VLA (83.0% real-robot) 对比，M²-VLA 笔记信息严重不足。MoL 像 MoE 变体，MSM 像 skill library——组合拳 novelty 存疑。
- **Rating**: `2` 👀 需看全文判断，但从 abstract 不像有 breakthrough。同期 VLA 论文显著更强。

### CF-VLA
- **Title**: CF-VLA: Efficient Coarse-to-Fine Action Generation for Vision-Language-Action Policies
- **Institutes**: 未记录（基于 abstract）
- **Source**: [link](http://arxiv.org/abs/2604.24622v2) 📄 arXiv   **📒 论文笔记**: [[2604-CFVLA]]
- **核心**: Flow-based VLA 推理慢，CF-VLA 两阶段解耦：coarse stage 学 endpoint velocity 条件后验作结构化初始化，fine stage 单步修正。NFE=2 即可工作。real-robot 83.0% success，latency -75.4%。
- **锐评**: Coarse-to-fine 非新概念，libra-VLA 同期类似思路。endpoint velocity 从哪来未澄清——数据集终点泛化存疑，预测误差传导fine stage。83.0%超MIP19.5pt、π₀.₅4.0pt数字太漂亮，但real-robot任务数、失败case分析缺失。标题"efficient"但只报告 latency，无 wall-clock time。novelty 不透明。
- **Rating**: `2` 👀 有价值数据点，但方法 novelty 和实验细节不够透明。

### Libra-VLA
- **Title**: Libra-VLA: Achieving Learning Equilibrium via Asynchronous Coarse-to-Fine Dual-System
- **Institutes**: Beihang University, AgiBot
- **Source**: [link](http://arxiv.org/abs/2604.24921v1) 📄 arXiv   **📒 论文笔记**: [[2604-LibraVLA]]
- **核心**: VLA action space 拆成离散 coarse bins + 连续 fine refinement，异步执行。发现倒 U 型曲线——N=10 bins 是性能甜点（"Libra point"）。LIBERO 97.2% avg success。
- **锐评**: 倒U型曲线是真 insight，N=10 最优是 empirical 无理论解释。LIBERO benchmark 天花板低，无 real-world 复杂场景。与 HybridVLA 对比基于30k steps 不公平。97.2%漂亮但benchmark本身简单，zero-shot 79.5%才是真考验。
- **Rating**: `2` 👀 倒U型+异步思路值得追踪，但 empirical finding 无理论支撑、实验有 cherry-pick 嫌疑。

### Odysseys
- **Title**: Odysseys: Benchmarking Web Agents on Realistic Long Horizon Tasks
- **Institutes**: Carnegie Mellon University
- **Source**: [link](http://arxiv.org/abs/2604.24964v1) 📄 arXiv   **📒 论文笔记**: [[2604-Odysseys]]
- **核心**: 首个长时域 web benchmark，200 个真实浏览任务在 live Internet 评估。Frontier 44.5% 成功率，Efficiency 仅 1.15%。Rubric-based evaluation 比 binary pass/fail 更细粒度，效率首次成为 first-class concern。
- **锐评**: 问题定义精准——直击 WebArena/WebVoyager 都是 static+short-horizon 痛点。Live Internet + long-horizon + efficiency 三维一体设计有诚意。但 44.5% 无对比无法判断 gap 来源。Efficiency 1.15% 是 agent 笨还是任务本身需要多步骤？Live Internet 双刃剑：真实但不可复现。annotation cost、agreement 缺失。
- **Rating**: `3` 🔥 问题定义精准直击痛点，benchmark 设计有诚意，值得深入。

### SnapGuard
- **Title**: SnapGuard: Lightweight Prompt Injection Detection for Screenshot-Based Web Agents
- **Institutes**: National University of Defense Technology; USTC; NUS; Zhejiang University  
- **Source**: [link](http://arxiv.org/abs/2604.25562v1) 📄 arXiv   **📒 论文笔记**: [[2604-SnapGuard]]
- **核心**: 针对截图型 Web Agent 的 prompt injection 检测。观察到恶意注入页面有两个特征——(1) 视觉梯度分布异常平滑（VSI），(2) 文本含行动诱导模式（APD）。OCR提取+梯度方差+规则匹配，F1=0.75，比GPT-4o prompt快8倍且无GPU开销。
- **锐评**: F1=0.75意味着25%漏检+误检，标题敢写"Accurate"纯属误导。VSI假设脆弱——攻击者加点纹理噪声就绕过。GPT-4o-prompt baseline是稻草人。APD是规则匹配，新攻击模式一来失灵。有价值insight是text-based defense在screenshot setting失效，但这只是观察不是贡献。
- **Rating**: `2` 👀 有特定场景价值（延迟敏感、资源受限），但检测精度不足以支撑"安全"claim。

### AgenticCache
- **Title**: AgenticCache: Cache-Driven Asynchronous Planning for Embodied AI Agents
- **Institutes**: Harvard University
- **Source**: [link](http://arxiv.org/abs/2604.24039v1) 📄 arXiv   **📒 论文笔记**: [[2604-AgenticCache]]
- **核心**: Embodied tasks 的 plan locality启发，用 cached plans 替代 per-step LLM calls。Runtime cache 查询 + background async updater。+22% success，-65% latency，-50% token。
- **锐评**: Plan locality observation 对但非新洞察。+22% baseline 是 naive per-step，意义有限。核心假设存疑——hard embodied tasks plan 变化剧烈，plan locality弱。蹭 world model 热点但实际是 efficiency optimization。缺失 cache hit rate、task complexity breakdown 等 ablation。
- **Rating**: `2` 👀 实用价值有，engineering contribution 可圈可点。但学术 novelty 有限，核心假设在 hard settings 存疑。

### SketchVLM
- **Title**: SketchVLM: Vision language models can annotate images to explain thoughts and guide users
- **Institutes**: College of William & Mary (推测)
- **Source**: [link](https://arxiv.org/abs/2604.22875) 📰 HF Daily ⬆️ 26   **📒 论文笔记**: [[2604-SketchVLM]]
- **核心**: Training-free 框架让 VLM 在图像上生成 SVG overlay 可视化推理过程。7 个 benchmark +28.5pp accuracy，annotation-text faithfulness >94%。卖点：不破坏原图、无需训练、跨模型通用。
- **锐评**: 本质是 prompt engineering + 坐标系 overlay 包装。+28.5pp 跟 image-editing baseline 比（后者破坏原图）是"比烂苹果好"。GPT-5 加 grid 后 error 飙到 99.34 说明"通用框架"对强模型有干扰。小模型 Qwen2.5-VL-7B 在复杂 drawing commands 拉胯，暴露对模型能力强依赖。>94% faithfulness 是最硬数字，证明视觉输出和文本一致。coordinate prompting + visual annotation 思路可迁移到 GUI grounding 验证。
- **Rating**: `2` 👀 方法是聪明的 prompt engineering 包装，但 faithfulness 验证和 non-destructive SVG 有实用价值。适合作为 baseline 参考。

### AutoResearchBench
- **Title**: AutoResearchBench: Benchmarking AI Agents on Complex Scientific Literature Discovery
- **Institutes**: Renmin University of China; AutoResearchBench Team
- **Source**: [link](https://arxiv.org/abs/2604.25256) 📰 HF Daily ⬆️ 23   **📒 论文笔记**: [[2604-AutoResearchBench]]
- **核心**: 科学文献发现 benchmark，Deep Research（找特定论文）+ Wide Research（收集满足条件论文组），基于 3M+ arXiv corpus。Claude-Opus-4.6 仅 9.39% accuracy、Gemini-3.1-pro 仅 9.31% IoU——与 BrowseComp 80%+ 形成对比，揭穿 frontier models 在 scientific search 上的假象 saturation。
- **锐评**: 问题定义精准，Deep/Wide dual paradigm 捕捉真实研究 workflow。full-text-first construction pipeline 严谨。数字够"毒"——<10%揭穿 BrowseComp"饱和"假象，scientific literature search 是 distinct capability frontier。但硬伤：18人作者团队分工不明，依赖自建 DeepXiv 平台闭环复现难。End-to-end 仅测 50 queries 样本量偏小。纯 benchmark 无 method innovation。Error analysis 偏定性缺 quantitative breakdown。
- **Rating**: `2` 👀 问题定义有洞察、construction 严谨、数字毒性强，但作者膨胀、无方法创新、闭环复现难。

### DRAGON
- **Title**: DRAGON: A Benchmark for Evidence-Grounded Visual Reasoning over Diagrams
- **Institutes**: Arizona State University, Adobe Research, University of Maryland
- **Source**: [link](http://arxiv.org/abs/2604.25231v1) 📄 arXiv   **📒 论文笔记**: [[2604-DRAGON]]
- **核心**: VLM diagram QA "答对≠真懂"。构建 evidence grounding benchmark：给定 diagram+question+answer，预测支持答案的 bounding boxes。11,664 实例，评测 8 个 VLM。
- **锐评**: 问题定义漂亮——从 accuracy 转向 evidence-grounded reasoning。但设计硬伤：given correct answer 再找证据是倒推验证非真实推理。6个数据集拼凑感明显。closed-source > open-weight 结论毫无惊喜。最核心问题未回答：evidence grounding 和 answer correctness 的 correlation。
- **Rating**: `2` 👀 方法有启发但设计硬伤——given answer 设定让 benchmark 从"测理解"降级为"测验证"。
