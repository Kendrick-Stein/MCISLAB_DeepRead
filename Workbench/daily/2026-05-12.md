### Workspace-Bench
- **Title**: Workspace-Bench 1.0: Benchmarking AI Agents on Workspace Tasks with Large-Scale Real-World Dependencies
- **Institutes**: SJTU, Alibaba
- **Source**: [link](https://arxiv.org/abs/2605.03596) 📰 HF Daily ⬆️ 7   **📒 论文笔记**: [[2605-WorkspaceBench]]
- **核心**: 提出 workspace 级 agent benchmark，含 5 角色、388 任务、20,476 文件（74 格式），显式标注文件依赖图并定义 6 维 workspace learning 能力。评估 28 种 agent 配置，最佳仅 68.7% pass rate（人类 80.7%），Heterogeneous File Understanding 和 Lineage Tracing 为核心瓶颈。五阶段 Workspace Learning 框架揭示了 L2 起 harness 贡献 > LLM 的关键洞察。
- **锐评**: Benchmark 设计诚意十足——74 文件类型 + 显式依赖图标注确实触及了真实 workspace 的复杂性。但 "dependency reasoning" 的测试深度存疑：任务是否真正要求 agent 理解文件间的语义/版本谱系，还是可通过 keyword 匹配 + 浅层推理绕过？Hard 任务仅占 33%，且人类基线 80.7% 暗示任务本身可能不够 challenging。Agent-as-a-Judge 的可靠性未充分讨论，judge model 偏差可能系统性影响评分。更关键的是，数据来源高度依赖字节跳动 Lark 平台的 154 个场景，外部效度存疑——其他企业 workspace 结构可能差异很大。五阶段框架有启发性，但 "singularity" 概念缺乏理论支撑，更多是 empirical observation。
- **Rating**: `3` 🔥 规模和工程投入突出，workspace learning 维度划分有诊断价值，但方法论创新有限，核心是 data contribution 而非 insight。

### Listwise Policy Optimization
- **Title**: Listwise Policy Optimization: Group-based RLVR as Target-Projection on the LLM Response Simplex
- **Institutes**: 未标注（作者 11 人）
- **Source**: [link](https://arxiv.org/abs/2605.06139) 📰 HF Daily ⬆️ 60   **📒 论文笔记**: [[2605-LPO]]
- **核心**: 揭示 group-based RLVR（GRPO 等）的隐式几何结构——每个方法都在 response simplex 上隐式定义 target distribution 并通过一阶近似投影。基于此提出 LPO，显式执行 target-projection：将 proximal RL objective 限制在 simplex 上，再通过精确 divergence minimization 投影 policy。理论保证单调改进、bounded/zero-sum/self-correcting gradients，实验声称 consistent improvement over baselines。
- **锐评**: 几何视角的 insight 有价值——把隐式结构显式化是方法论上的贡献。但 claim "monotonic improvement" 需要看证明细节；divergence 解耦的理论优势在实验中是否体现未知。institution 缺失影响可信度。核心问题：LPO 的 target 和 GRPO 的隐式 target 是否本质不同？如果是同一 target，LPO 的优势仅在于"精确投影 vs 一阶近似"，这可能只是数值优化层面的改进而非方法创新。需获取全文验证实际性能差距和 ablation。
- **Rating**: `3` 🔥 方法论 insight 有价值，但实际贡献需验证；与现有 RLVR 方法的关系需更清晰阐述。

### AEM
- **Title**: AEM: Adaptive Entropy Modulation for Multi-Turn Agentic Reinforcement Learning
- **Institutes**: Baidu, Tsinghua University, Fudan University
- **Source**: [link](https://arxiv.org/abs/2605.00425) 📰 HF Daily ⬆️ 16   **📒 论文笔记**: [[2605-AEM]]
- **核心**: 提出一种 supervision-free 的 response-level credit assignment 方法。核心理论发现：entropy drift 由 advantage 与 relative surprisal 的交互决定。AEM 用 response-level entropy proxy rescale advantage，实现从 exploration 到 exploitation 的自适应过渡。在 ALFWorld/WebShop 上 GRPO+1.5B 提升 8.8%，SWE-bench-Verified 上 DeepSWE+32B 提升 1.4%。
- **锐评**: 理论扎实——从 Fisher-Rao geometry + natural gradient 推导 entropy drift 公式，数学基础清晰，不是凭经验调参。Supervision-free 是亮点：PRM 需要 annotation，value function 需要额外模型，AEM 完全 intrinsic。但 entropy proxy 是近似：用 token-level entropy sum 代替 response surprisal，虽然是 Doob decomposition 的 predictable 部分，但理论上只是部分逼近。缺乏与 PRM 的 head-to-head 对比——只对比了 RL baselines，未展示 supervision-free 相比 PRM 的 trade-off（性能/成本）。Response-level 假设的适用边界未充分讨论，某些 streaming feedback 或 step-level reward 场景可能不适用。
- **Rating**: `3` 🔥 深度理论分析 + supervision-free 设计 + 跨 benchmark 提升，但缺少与 PRM 对比且 entropy proxy 是近似，属于"有参考价值"而非"必读"。

### Don't Click That
- **Title**: Don't Click That: Teaching Web Agents to Resist Deceptive Interfaces
- **Institutes**: Renmin University of China, Ant Digital Technologies (Ant Group)
- **Source**: [link](http://arxiv.org/abs/2605.09497v1) 📄 arXiv   **📒 论文笔记**: [[2605-DUDE]]
- **核心**: DUDE 是首个针对 VLM Web Agent 的欺骗防御框架。通过两阶段方法——Stage 1 用 asymmetric reward（欺骗点击惩罚 10x）训练 evaluator，Stage 2 将失败案例迭代蒸馏为可迁移的 textual guidance。在 RUC benchmark（1,407 scenarios）上减少 53.8% 欺骗易感性，同时保持任务性能，且经验可迁移到 closed-source models（GPT-4o）。
- **锐评**: Defense 有效，但验证不够完整。这篇与 Decepticon、TrickyArena 等攻击论文形成了呼应——attack 侧已经证明 agent 高度脆弱（70%+ deception rate），defense 侧终于有了系统性方案。核心创新在于两点：(1) asymmetric penalty 的直觉正确——批准欺骗性点击的后果远比误拒合法点击严重；(2) experience summarization 实现 deployment-time improvement without parameter updates。但弱点也很明显：只在单次点击决策上评估，未在完整 WebArena/OSWorld 任务流程上验证；deception taxonomy 仅四种，真实世界 dark patterns 更多样；summarizer 依赖外部模型增加部署成本。更重要的是，它只防御"已知类型"的 deception，对 novel attack patterns 的泛化能力存疑。
- **Rating**: `3` 🔥 问题重要（Web Agent 安全部署的关键缺失环节），方法有创新点（asymmetric RL + prompt-level experience accumulation），但实验验证不够深入，完整闭环需在端到端 benchmark 上验证。

### GazeVLM
- **Title**: GazeVLM: Active Vision via Internal Attention Control for Multimodal Reasoning
- **Institutes**: [未获取全文，待补充]
- **Source**: [link](http://arxiv.org/abs/2605.07817v1) 📄 arXiv   **📒 论文笔记**: [[2605-GazeVLM]]
- **核心**: GazeVLM 将人类主动视觉原理引入 VLM，通过 internal attention control 机制让模型自主决定 "where to look next"，实现细粒度视觉推理。区别于传统 VLM 被动处理整张图像的方式，模型能动态聚焦相关区域进行任务驱动的视觉分析。[注：未获取全文，基于早期版本信息整合]
- **锐评**: 方向价值——将 active vision 引入 VLM 是正确的研究方向，人类视觉确实通过主动 gaze 实现高效感知，而不是一次性处理全图。这个 insight 有价值。机制存疑——internal attention control 的具体实现细节不明。"让模型决定 where to look next" 听起来像 reasoning-guided attention，但这和现有的 attention sink、token pruning 等方法有何本质区别？需要看具体实现。关键问题：gaze trajectory 如何学习？是 supervised（需要 gaze ground truth）还是 RL-based？如果是 supervised，数据从哪来？如果是 RL，reward 如何设计？这些决定了方法的 scalability。版本差异：2605.07817 标题强调 "Internal Attention Control for Multimodal Reasoning"，早期版本标题为 "Fine-Grained Gaze Optimization"，两者侧重点可能不同，需要对比。局限待验证：计算开销是否合理？相比 crop + multi-scale 等简单方案是否有明显优势？
- **Rating**: `3` 🔥 方向有价值，active vision + VLM 的结合点正确，但机制细节和实验验证待获取全文后评估。

### OA-WAM
- **Title**: OA-WAM: Object-Addressable World Action Model for Robust Robot Manipulation
- **Institutes**: 未在 abstract 中明确，作者含 Xiao-Ping Zhang（疑 Western University）
- **Source**: [link](https://arxiv.org/abs/2605.06481) 📄 arXiv   **📒 论文笔记**: [[Papers/2605-OAWAM]]
- **核心**: 将 World Action Model 的 holistic world representation 分解为 N+1 个 object-addressable slots（persistent address + time-varying content），通过 address-only key routing 在每层重置实现对象级可寻址性，同一 forward pass 内联合预测世界状态和 action chunk。LIBERO 97.8%、SimplerEnv 79.3% 匹配强 baseline，LIBERO-Plus geometric axes 达 SOTA。Causal slot-intervention 测试显示 swap-binding cosine 0.87，远超 holistic baseline 的 0.09。
- **锐评**: Address-content 分离的直觉确实优雅——"你是谁"和"你现在什么状态"解耦，每层重置 address slice 不加 token 也干净。但 0.87 的 swap-binding cosine 说明的是"表示层可寻址性做得好"，不是"任务做得更好"——LIBERO/SimplerEnv 只是 match baseline，不是 beat baseline。说白了，你的 slot 编码确实知道哪个对象是哪个，但这并没有转化为显著的 manipulation 性能优势。N+1 slot 假设预设了已知对象数量，在真实厨房里你连有几个碗都不确定。此外，object-centric representation 的经典病——slot collapse、动态对象数量、遮挡恢复——在 abstract 中完全没提。方法有巧思，但离"robust robot manipulation"的 claim 还差一个数量级的验证。
- **Rating**: `3` 🔥 object-addressable 设计有参考价值，但 benchmark 增量有限，robustness 的 claim 需要更强的证据支撑。

### StraTA
- **Title**: StraTA: Incentivizing Agentic Reinforcement Learning with Strategic Trajectory Abstraction
- **Institutes**: Shanghai AI Laboratory, University of Oxford, Shanghai Jiao Tong University
- **Source**: [link](https://arxiv.org/abs/2605.06642) 📰 HF Daily ⬆️ 21   **📒 论文笔记**: [[Papers/2605-StraTA]]
- **核心**: 当前 agentic RL 的核心瓶颈是 pure reactive 策略——agent 只看当前 observation 做决策，exploration 和 credit assignment 都受限。StraTA 的解法很直觉：在 GRPO 上加一层 trajectory-level strategy，先从初始状态生成一个 compact plan，再 conditioning 后续 action，用 hierarchical rollout 联合训练。ALFWorld 93.1%、WebShop 84.2%、SciWorld 63.5%，三个 benchmark 都有提升。
- **锐评**: Idea 本身不新鲜——"think before you act" 在 agent 领域已经是老生常谈，LLM-based agent 本来就可以用 CoT/plan 来做 trajectory-level reasoning，StraTA 只是把这个过程 formalize 进了 RL training loop。关键问题是：(1) strategy 的 failure mode 没讨论，bad plan 会 cascading 到整个 trajectory；(2) "outperforming frontier closed-source models" on SciWorld 是不公平对比——GPT-4/Claude 没有在 SciWorld 上做 RL fine-tuning，拿 RL-tuned agent 比 zero-shot LLM 是 strawman；(3) strategy 到底是什么形式？自然语言？latent embedding？这决定了方法的可解释性和可迁移性，abstract 里故意回避了这个问题。整体感觉是 "把 planning module 塞进 GRPO pipeline" 的 incremental work，包装得不错但 insight 有限。
- **Rating**: `3` 🔥 问题真实、方法合理、结果尚可，但核心 idea（hierarchical planning + RL）不算新，且缺乏对 strategy 质量和 failure mode 的深入分析。

### When to Trust Imagination
- **Title**: When to Trust Imagination: Adaptive Action Execution for World Action Models
- **Institutes**: Southern University of Science and Technology, The University of Hong Kong
- **Source**: [link](https://arxiv.org/abs/2605.06222) 📰 HF Daily ⬆️ 39   **📒 论文笔记**: [[Papers/2505-WhenToTrust]]
- **核心**: 提出 FFDC verifier 将 WAM 执行从"固定 chunk"转向"可靠性感知控制"，通过实时验证 prediction-observation consistency 动态决定何时继续执行、何时重规划。在 RoboTwin 上减少 69% 推理调用、34% 执行时间，成功率反而提升 2.5%；real-world 提升 35%。核心洞察是利用 WAM 独有的"预测未来视觉观测"能力做自验证。
- **锐评**: "when to trust" 这个问题的回答相当清晰——通过 FFDC 联合推理 predicted actions + visual dynamics + real observations + instructions，输出"是否可信任"的判断。设计合理之处在于抓住了 WAM 的独特属性（其他 VLA/diffusion policy 没有 visual prediction），而非依赖 action-side uncertainty。但有一点值得追问：FFDC 本身是持续运行的，它减少了 WAM 调用但引入了 verifier overhead，这个 trade-off 的边界在哪里？另外 binary supervision 的局限性文中已承认，这可能是泛化瓶颈。整体而言，从"选 chunk size"转向"verify trust"的视角转换很有启发。
- **Rating**: `3` 🔥 问题定义精准、方法设计有洞察，但技术细节（FFDC 架构、训练数据构造）因未获取全文留有悬念，值得后续跟进。

### ARIS
- **Title**: ARIS: Autonomous Research via Adversarial Multi-Agent Collaboration
- **Institutes**: 未标注机构（从 GitHub 看似个人/小团队项目）
- **Source**: [link](https://arxiv.org/abs/2605.03042) 📰 HF Daily ⬆️ 107   **📒 论文笔记**: [[2605-ARIS]]
- **核心**: 单 agent 做长期科研任务不可靠，会产生"看似合理但无证据支撑"的结果。解法是跨模型对抗协作：executor 推进任务，来自不同模型家族的 reviewer 独立审计并要求修订。三层架构：Execution（65+ skills）、Orchestration（5 个 workflow）、Assurance（三阶段证据审计）。案例显示 8 小时自动运行后 reviewer 分数从 5.0 提升到 7.5。
- **锐评**: 硬伤明显——全文零定量评估。作为系统论文，没有一个 benchmark、没有对比实验、没有消融。"cross-model adversarial 比同模型自纠更好"是核心 claim，但作者根本没验证——信了再说。Claim 与证据严重错配：Assurance Layer 设计了三层审计 cascade，看起来很 fancy，但审计到底 catch 了多少问题？误报率多少？漏报率多少？全黑盒。Overnight case study 里"移除了 unsupported claims"——移除了多少？移除之后对吗？不知道。本质区别存疑：和 AI Scientist、Agent Laboratory 比，差异点确实是 cross-model adversarial。但 reviewer 本身的可靠性问题完全没讨论。如果 executor 和 reviewer 都有盲区怎么办？"谁来评判评判者"是古老问题，论文假装不存在。Appendix E 有意思：作者提出了合理的 benchmark 协议（12+ papers, 5 conditions, 3 blinded raters）——但为什么没做？
- **Rating**: `2` 👀 想法有洞察，工程量大，但作为研究论文，核心假设未验证是硬伤。更像一个 polished prototype demo，等有实验数据再谈价值。

### MolmoAct2
- **Title**: MolmoAct2: Action Reasoning Models for Real-world Deployment
- **Institutes**: Allen Institute for AI, University of Michigan
- **Source**: [link](https://arxiv.org/abs/2605.02881) 🔥 HF Trending ⬆️ 273   **📒 论文笔记**: [[2605-MolmoAct2]]
- **核心**: 全开源 VLA 系统，五个模块化组件：embodied reasoning 专用 VLM backbone（Molmo2-ER）、三个新数据集（含最大开源双臂数据集 BimanualYAM）、开源 action tokenizer（OpenFAST）、flow-matching action expert、自适应深度推理。Zero-shot 真实世界部署大幅超越 Pi0.5（DROID +41.9%），LIBERO fine-tune 后 98.1% SOTA。
- **锐评**: "最全面开源 VLA 研究" 的 title 听起来响亮，但核心架构 novelty 几何？Flow-matching + per-layer KV conditioning 都是现有技术的排列组合。所谓 "系统贡献" 更像是工程堆叠而非方法论突破。Claim 与证据不匹配：宣称 "zero-shot 部署能力强"，但 MolmoSpaces Open 任务仅 9.5%（Pi0.5 是 22.7%）——这种选择性报告令人起疑。Stack blocks 20%、Block in box 33.3% 对比 Pi0 的 6.7%/90.0%，这种退化数据藏在附录里就当不存在？与其说是 "action reasoning model"，不如说是 "embodied data + compute 的胜利"。9,200+ H100 hours 的训练成本，这个复现门槛让 "开源" 二字显得讽刺——代码开源了，但有几家机构跑得起？
- **Rating**: `2` 👀 系统工程扎实，数据贡献有价值，但 "突破性" 有限，架构创新不足，zero-shot 能力被选择性强调。开源社区的 "工程范本"，不是研究范本。

### OpenSearch-VL
- **Title**: OpenSearch-VL: An Open Recipe for Frontier Multimodal Search Agents
- **Institutes**: 上海科技大学等（见论文原文）
- **Source**: [link](https://arxiv.org/abs/2605.05185) 📰 HF Daily ⬆️ 95   **📒 论文笔记**: [[2605-OpenSearchVL]]
- **核心**: 提出首个完全开源的 Multimodal Deep Search Agent 训练配方，包括 36k SFT + 8k RL 轨迹数据、工具环境（检索+增强+OCR）和 Fatal-Aware GRPO 算法。核心创新是通过 fatal step masking 和 one-sided advantage clamping 解决 cascading tool failure 问题，在 7 个 benchmark 上平均提升 10+ 分，开源 32B 模型超越 Gemini-2.5-Pro。
- **锐评**: Fatal-Aware GRPO 设计确实聪明——one-sided clamping 让有效前缀不被无效惩罚，消融显示比 hard masking 多 4.1 分，证明这个改进不是花架子。Source-Anchor Grounding 强制 agent 从视觉入口出发沿关系链推理，杜绝 shortcut retrieval，消融 -11.5 分证明其必要性。但硬伤："开源配方依赖闭源评估"是自相矛盾的——用 GPT-4o judge accuracy、GPT-5.4 judge process quality，评分成本高且本身不可复现。更关键的是，query-quality reward 仅评估文本 query，视觉操作的质量完全没纳入奖励信号，reward 函数存在结构性盲区。与 SOLAR-RL 的 first failure point detection 思路相似但技术实现不同——OpenSearch-VL 用 masking + clamping，SOLAR-RL 用 validity detection + reward alignment。
- **Rating**: `2` 👀 开源完整性值得肯定，fatal-aware GRPO 是有效的增量改进，但"开源配方依赖闭源评估"的悖论削弱了价值主张。适合作为 agentic RL 的参考实现，但不要指望它解决根本性的 credit assignment 难题。

### Skill1
- **Title**: Skill1: Unified Evolution of Skill-Augmented Agents via Reinforcement Learning
- **Institutes**: 未获取（仅基于 abstract）
- **Source**: [link](https://arxiv.org/abs/2605.06130) 📰 HF Daily ⬆️ 68   **📒 论文笔记**: [[2605-Skill1]]
- **核心**: 提出 skill library 维护的三种耦合能力（selection/utilization/distillation），用单一策略统一训练，通过奖励信号频率分解（低频→selection credit，高频→distillation credit）实现 credit assignment。在 ALFWorld 和 WebShop 上超越 baselines。
- **锐评**: "奖励信号频率分解"听着很 fancy——但一个 task-outcome 信号凭什么能稳定地区分出 selection 和 distillation 的贡献？低频成分给 selection、高频给 distillation，这假设是否过于理想化？ALFWorld + WebShop——经典"挑简单的打"组合。这些环境的 skill library 维护真的能体现复杂场景的挑战吗？"co-evolution" 的核心 claim：训练动态确认三种能力协同进化——但协同进化和各自单独训练有什么本质区别？是否只是 multi-task learning 换了个名字？Abstract 里 skill representation、library 检索机制都是黑盒——魔鬼在细节，而这些恰恰是决定方法能否 scale 的关键。
- **Rating**: `2` 👀 想法有趣，reward decomposition 的视角值得玩味；但证据单薄，需要看全文才能判断是真 insight 还是 fancy packaging。

### MiniCPM-o 4.5
- **Title**: MiniCPM-o 4.5: Towards Real-Time Full-Duplex Omni-Modal Interaction
- **Institutes**: OpenBMB, Tsinghua University
- **Source**: [link](https://arxiv.org/abs/2604.27393) 📰 HF Daily ⬆️ 64   **📒 论文笔记**: [[2604-MiniCPMo45]]
- **核心**: 9B 参数的全模态模型，提出 Omni-Flow 统一流式框架，将视觉/音频/文本输入输出沿共享时间轴对齐，实现 real-time full-duplex 交互。架构上 LLM backbone (Qwen3-8B) 只生成 text token（3-4 tokens/s），语音由轻量 Llama decoder (~0.3B) + CosyVoice flow-matching 合成，避免直接生成 speech token 带来的效率退化。TAIL 策略自适应对齐语音播放与交互时间线。<12GB RAM 边缘部署。
- **锐评**: Full-duplex 的 "proactive behavior" 本质上靠预定义场景的 instruction data 驱动，真正的开放环境主动行为仍是空中楼阁。1.0s chunk size 的延迟在实时翻译等场景下捉襟见肘。论文大篇幅对比同 scale 模型（Qwen3-Omni-30B），却巧妙回避了与 GPT-4o 等闭源模型全双工能力的正面比较——这不是巧合。Omni-Flow 的时间轴对齐思想本身是好的，但 ablation table 暴露了一个尴尬事实：chunk size 从 1.0s 降到 0.2s 时模型几乎崩溃（AlpacaEval 3.56→1.22），说明所谓的 "real-time" 其实很脆弱。
- **Rating**: `2` 👀 Omni-Flow 框架设计有参考价值，但 full-duplex 的实际成熟度被高估了，9B 做边缘部署的工程亮点大于学术 novelty。

### WebTrap
- **Title**: WebTrap: Stealthy Mid-Task Hijacking of Browser Agents During Navigation
- **Institutes**: Wuhan University, George Mason University, City University of Hong Kong
- **Source**: [link](http://arxiv.org/abs/2605.08310v1) 📄 arXiv   **📒 论文笔记**: [[2605-WebTrap]]
- **核心**: 提出 WebTrap，一种针对 browser agent 的中间任务劫持攻击。核心思路是"寄生式"目标融合——利用 LLM 生成与用户任务上下文对齐的恶意网页内容，让 agent 在执行攻击目标后无缝恢复原始任务，实现隐蔽劫持。在 WASP-e 和 InjecAgent-e 上 ASR 分别达 82.2% 和 65.7%，远超已有方法；在 Amazon/Best Buy/Walmart 真实网站上达到 100% ASR。当前 prompt-level 防御几乎全部失效。
- **锐评**: 攻击设计的"寄生"思路确实比粗暴的目标替换高明——让 agent 完成攻击后继续干活，用户根本察觉不到，这是对 prompt injection 的一个有意义的升级。但问题在于：(1) 内容生成器绑定 GPT-4o，连自己都不换模型试一下？开源模型 ASR 只有 42.6% 这个数据被一笔带过，明明是最有信息量的发现——为什么开源模型更鲁棒？不分析一下？(2) 防御方实验选择性太强，测了一堆 prompt-level 防御然后宣布"防御无效"，这跟射箭画靶有什么区别？action verification、sandbox isolation、human-in-the-loop 这些架构级防御一个没测。(3) 真实网站 100% ASR 听起来吓人，但攻击者需要**控制目标网页内容**这个前提被轻描淡写——这不就是传统 web phishing 的变体吗？跟"browser agent 安全漏洞"的 framing 有偷换概念之嫌。
- **Rating**: `2` 👀 攻击思路有新意但 defense analysis 严重偏科，真实威胁场景被夸大，更像一篇"攻方视角的 demo paper"而非严谨的安全分析。

### Mobile World Model for GUI Agents
- **Title**: How Mobile World Model Guides GUI Agents?
- **Institutes**: Tsinghua University, Nanyang Technological University
- **Source**: [link](http://arxiv.org/abs/2605.10347v1) 📄 arXiv   **📒 论文笔记**: [[2605-MobileWorldModelGUI]]
- **核心**: 系统比较 delta text / full text / diffusion image / renderable code 四种模态的 mobile world model，发现 renderable code 分布内保真度高且能提供多模态训练监督，text-based feedback 在 OOD 线上执行更鲁棒。World model 生成的轨迹可作为训练数据提升 agent 性能，但对 overconfident agent 做 post-hoc self-reflection 效果有限——world model 更适合做 prior perception 而非 test-time verifier。
- **锐评**: 本质是一篇 engineering ablation 而非方法论创新。"四种模态比较"的框架清晰但缺乏深度——renderable code 的可渲染假设在非标准 UI（游戏、动态 Web）上泛化性存疑，diffusion image 方案的计算开销和延迟在 mobile 场景下是否 practical 完全没讨论。Finding 3（post-hoc reflection 无效）的结论下得太快：可能是 reflection 策略设计粗糙，而非 world model 本身不适合做 verifier。此外，AITZ/AndroidControl/AndroidWorld 三个 benchmark 都是相对受限的 setting，真正 OOD 的 wild 场景未触及。一篇扎实但 uninspiring 的实证工作，对后续研究有参考价值但不会改变方向。
- **Rating**: `2` 👀 有参考价值的系统比较，但缺乏方法创新和深层 insight，不会改变 anyone's research direction。
