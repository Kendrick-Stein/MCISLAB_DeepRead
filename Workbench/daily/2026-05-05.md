## 论文点评

### MolmoAct2
- **Title**: MolmoAct2: Action Reasoning Models for Real-world Deployment
- **Institutes**: Allen Institute for AI, University of Michigan
- **Source**: [link](https://arxiv.org/abs/2605.02881) 🔥 HF Trending ⬆️ 176 **📒 论文笔记**: [[Papers/2605-MolmoAct2]]
- **核心**: Allen AI 的全开源 VLA 系统，五大维度系统性升级：Molmo2-ER backbone（3.3M embodied reasoning 语料，4B 模型在 13 个 ER benchmark 上平均 63.8% 超越 GPT-5 和 Gemini ER-1.5）、三个新数据集（含最大开源双臂数据集 BimanualYAM 720h）、OpenFAST 跨 embodiment action tokenizer、flow-matching DiT action expert 通过 per-layer KV-cache conditioning 接入 VLM、MolmoAct2-Think 自适应深度推理。在 7 个仿真与真实 benchmark 上全面超越 Pi0.5，真实世界 DROID 达 87.1%（Pi0.5 仅 45.2%），LIBERO fine-tune 达 98.1%。
- **锐评**: 这是一个 engineering 系统整合贡献，architecture novelty 有限——flow-matching + KV conditioning 在 Pi0/Pi0.5 中已有先例，OpenFAST 本质上是 FAST 的开源复现。亮点在于：(1) 全链路开源程度罕见，从数据到 tokenizer 到权重到训练代码全部释放；(2) BimanualYAM 数据集用 <$6K 采集 720h 双臂数据，社区贡献大；(3) 真实世界跨三种硬件验证扎实。但问题也很明显：总训练成本 ~9,200 H100 GPU hours，复现门槛不低；zero-shot 在 MolmoSpaces Open 任务仅 9.5%（Pi0.5 为 22.7%），说明泛化仍有脆点；SO-100/101 真实世界部分任务明显退化（Stack blocks 20% vs Pi0 67%），社区数据质量筛选仍不够；LIBERO 上与闭源 GR00T N1.7 差距极小（98.1% vs 97.0%），增量有限。推理延迟优化提到了 CUDA Graph 但缺少具体数字对比。
- **Rating**: `3` 🔥 全链路开源的系统工程范本，数据和 backbone 贡献突出，但核心架构 novelty 有限，zero-shot 泛化仍脆，更适合作为 VLA 社区的基础设施而非方法论突破。

### T2PO
- **Title**: T^2PO: Uncertainty-Guided Exploration Control for Stable Multi-Turn Agentic Reinforcement Learning
- **Institutes**: UCLA, Amazon
- **Source**: [link](https://arxiv.org/abs/2605.02178) 📰 HF Daily ⬆️ 4 **📒 论文笔记**: [[Papers/2605-T2PO]]
- **核心**: 多轮 RL 训练崩溃的根源不是 credit assignment 而是低效探索。T²PO 在 token-level 监控 uncertainty 动态触发 thinking intervention，在 turn-level 识别低信息量交互并动态重采样。在 WebShop/ALFWorld/SearchQA 上验证训练稳定性和性能提升。与 ProxMO（credit assignment）和 SOLAR-RL（rollout 重建）正交互补。
- **锐评**: 从探索效率角度切入多轮 RL 不稳定性是正确的方向——现有 credit assignment 方法（ProxMO、SOLAR-RL）假设 rollout 本身是高质量的，但实际训练中大量 rollout 在"死胡同"turn 上浪费计算。Token-level uncertainty 监控 + turn-level resampling 的双层设计有层次感。但 baseline 选择偏弱，主要对比标准 GRPO；与 ProxMO/SOLAR-RL 的直接对比缺失，难以判断两条路线的相对收益。Uncertainty threshold 的 per-task 敏感性未讨论。代码开源是加分项。
- **Rating**: `2` 👀 探索效率视角有价值，与 credit assignment 方法正交，但实验对比不够充分，缺乏与最新 concurrent work 的直接对比。

### WindowsWorld
- **Title**: WindowsWorld: A Process-Centric Benchmark of Autonomous GUI Agents in Professional Cross-Application Environments
- **Institutes**: HITsz-TMG
- **Source**: [link](https://arxiv.org/abs/2604.27776) 📰 HF Daily ⬆️ 9 **📒 论文笔记**: [[Papers/2604-WindowsWorld]]
- **核心**: 面向跨应用专业工作流的 GUI Agent benchmark，通过多 agent 生成管线 + 16 个职业 persona 构建 181 个任务（78% 多应用），覆盖 4 个难度等级和 17 个桌面应用。核心创新是引入过程级 checkpoint 评估（S_int，每任务 4.97 个 checkpoint），而非仅看终态。所有 agent 在多应用任务上 S_final 不超过 21%（Gemini-3-flash, Hybrid 最佳），step-matched 对比证明跨应用上下文切换（而非步数长度）是独立瓶颈——L1 S_final 46% vs L2 仅 14%。
- **锐评**: 过程级评估设计确实比 OSWorld 精细，step-matched 跨应用难度分析是真正有价值的 insight。但 181 个任务规模偏小（L4 仅 12 个），统计可信度存疑；缺乏 human performance baseline 是硬伤——没有锚点就无法判断 20% 是 agent 差还是任务定义有问题。GPT-5.2 的灾难性失败（S_final ≈ 0）非常可疑，需排查是否 action space 映射问题而非模型能力缺陷。VLM judge 全依赖 Qwen3-VL-Plus 单一模型，judge 偏差无法交叉验证。应用覆盖严重偏科（Excel 73 + Thunderbird 72 占了 80% 任务），VLC 仅 1 个任务，多媒体场景形同虚设。作为 2026 年的 benchmark 未评估 MCP 工具集成是明显盲点。
- **Rating**: `2` 👀 过程级评估思路不错，但规模、human baseline 缺失、judge 单点依赖限制了 benchmark 的说服力。

### GLANCE
- **Title**: What You Think is What You See: Driving Exploration in VLM Agents via Visual-Linguistic Curiosity
- **Institutes**: HKUST, UESTC
- **Source**: [link](http://arxiv.org/abs/2605.03782v1) 📄 arXiv **📒 论文笔记**: [[Papers/2605-GLANCE]]
- **核心**: GLANCE 将 VLM agent 的语言预测与视觉现实之间的跨模态差异转化为内在好奇心信号，驱动 sparse-reward 环境下的探索式 RL。核心机制是用 momentum target encoder 编码"看到什么"，用 VLM 的 CoT prediction token 隐状态表征"想什么"，两者之间的 normalized MSE 作为 curiosity reward。为防止 projector 过拟合导致 curiosity drain，引入 Curriculum Exploration 机制周期性重置 projector 权重。在 5 个 agentic 任务上用 Qwen2.5-VL-3B 超越了 VAGEN 等 exploitation-based 方法。
- **锐评**: 概念有吸引力——"用语言预测驱动视觉探索"比纯视觉 novelty 更有语义意义——但执行层面有明显短板。第一，只用 3B 模型，完全没验证 scaling behavior，大模型的语义表征天然更强，projector 过拟合问题可能自然缓解，也可能加剧，缺乏分析。第二，Curriculum Exploration 的 projector 周期性 re-init 是典型的 workaround，暴露了 cross-modal alignment 训练的根本不稳定性，但论文没有深入分析为什么 projector 这么容易 collapse。第三，baseline 选择偏弱，主要是 exploitation-based 方法（VAGEN），缺少与语言空间 curiosity 或 recent exploration 方法的对比。第四，5 个任务都是相对标准的 agentic benchmark，缺乏 failure case 分析——跨模态 curiosity 何时会误导探索？当语言预测本身错误时，curiosity signal 会不会把 agent 引向死胡同？
- **Rating**: `2` 👀 跨模态 curiosity 的 idea 值得关注，但 3B-only 评估、workaround 式的训练稳定方案、以及缺乏 failure analysis，使其尚不足以改变 VLM agent exploration 的实践方式。

### WorkspaceBench
- **Title**: Workspace-Bench 1.0: Benchmarking AI Agents on Workspace Tasks with Large-Scale File Dependencies
- **Institutes**: SJTU, Alibaba
- **Source**: [link](http://arxiv.org/abs/2605.03596v1) 📄 arXiv **📒 论文笔记**: [[Papers/2605-WorkspaceBench]]
- **核心**: 现有 agent benchmark 的文件类型和依赖结构过于单一，无法反映真实办公 workspace 的复杂性。Workspace-Bench 构建了 5 个角色、388 个任务、20,476 个文件（74 种格式）的大规模 benchmark，显式标注文件依赖图并提出 6 维 workspace learning 分类。28 种 agent 配置评估显示最佳 agent（OpenClaw + Opus-4.7）仅 68.7% rubrics pass rate，远低于人类 80.7%，且在 Heterogeneous File Understanding 和 Lineage Tracing 上存在系统性瓶颈。
- **锐评**: 数据规模（74 文件类型、>2500h 标注时间）是实打实的工程贡献，甩开 TheAgentCompany（<20 文件类型）一个量级。但数据来源高度绑定字节跳动 Lark 平台的 154 个内部场景，外部效度存疑——换一家公司 workspace 结构可能完全不同。Agent-as-a-Judge 用 Seed-2.0-Lite 打分 7,399 条 rubrics，但 judge 自身偏差和标注一致性完全没有讨论，这让所有百分比数字的可信度打折扣。五阶段框架中 L2 "orchestration singularity"（harness > LLM）和 L3 "capability singularity" 的概念有启发性，但更像是 post-hoc 归纳而非有理论支撑的预测。本质上这是一篇 engineering-heavy 的 benchmark paper，核心贡献在于数据和评估框架，方法论创新有限。
- **Rating**: `2` 👀 规模和细粒度评估有参考价值，但数据来源局限、judge 可靠性未验证，且 benchmark paper 本身缺乏方法论突破。

### StateVLM
- **Title**: StateVLM: A State-Aware Vision-Language Model for Robotic Affordance Reasoning
- **Institutes**: University of Hamburg
- **Source**: [link](http://arxiv.org/abs/2605.03927v1) 📄 arXiv **📒 论文笔记**: [[Papers/2605-StateVLM]]
- **核心**: 提出 Auxiliary Regression Loss (ARL)，在 VLM fine-tuning 时用 box decoder 的 L1+GIoU loss 辅助训练，推理时回归标准 sequence prediction，零额外开销。配套贡献 OSAR benchmark（1,172 scenes，Stable Diffusion + Blender 生成的厨房场景）评估 object-state affordance reasoning。ARL 在 RefCOCO 系列上平均提升 1.6%，在 OSAR 上提升 5.2%，并将异常输出率从 33.36% 降至 0%。
- **锐评**: ARL 的核心思想——在 autoregressive VLM 上加辅助回归 loss——并不新鲜，DETR 系列和 NExT-Chat 早已采用类似策略，本文本质上是将其适配到 MiniCPM-V 上。RefCOCO 上 1.6% 的提升在 REC 领域属于噪声级别，且未超越 2023 年的 Shikra。OSAR benchmark 是本文最大的隐患：Stable Diffusion 生成的非真实数据、仅 1,172 scenes 的规模、且 ARL+LoRA 在 object detection 上反而低于 CLM-only（与 REC 结论矛盾），都削弱了结论的可信度。作者自己承认"significant gap remains between current performance and practical deployment"。
- **Rating**: `1` 💤 方法增量有限，合成 benchmark 可信度不足，与已有工作差异不大。
